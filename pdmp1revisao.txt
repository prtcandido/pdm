Hook (ver exemplohook.zip)
====

const [nome,setNome] = useState("Joao")

Para que serve o useState?

Resp.
Para preservar dados no componente (estado) mesmo que  
o componente seja renderizado

O que ocorre com o componente se um useState for modificado?

Resp.
O componente será novamente renderizado (código reexecutado)

Como alterar o valor do useState "nome" para "Maria"
Resp.:
setNome("Maria")

-----------

Qual é diferença principal entre useState e useRef?
Mudanças no useRef não causam a renderização do componente.

----------
Caso 1
useEffect( função , [] )

Dado o useEffect acima em um componente, quando a função será executada? 

Apenas 1 vez na carga do componente.

Caso2
useEffect( função , [a,b,c] )

Dado o useEffect acima em um componente, quando a função será executada? 

Na carga do componente e se houver mudança em a,b ou c.

Sobre o useQuery

Qual a vantagem do seu uso em relação a progração com useState e
useEffect?

Resp.: Controles automáticos de carregados dos dados e manutenção
automática de cache, facilita o tratamento de erros e resultados vazios.

------

Qual a diferença useQuery para useMutation?

Resp.:
useQuery é para consultas
useMutation é para delete, insert e update

Após atualizar, inserir ou excluir itens na API, 
qual operação é necessária

para o usuário?

Resp. Invalidar os caches.

O que é o Laravel?

Framework PHP para aplicação Web MVC e API-REST.

--------

Descreva os componentes criados pelo comando abaixo:

php artisan make:model Produto --controller --migration

Model: classes de objetos com os dados do negócio. 
       objetos persistentes.

Qual padrão do Larevel para nome de tabela? Nome do model no plural.

Controller: classes onde estão as funções associadas as requisições
       (rotas) dos clientes

Migration: código PHP para modificar a estrutura do banco de dados
        cria tabela, elimina tabela , adiciona coluna
        , adiciona restrição (chave estrangeira, por exemplo)

Rotas de API

Em qual arquivo são definidas as rotas de API?

routes\api.php

Quais são as rotas para produtos?

Consultando todos os produtos
GET
localhost:8000/api/produtos
Associada a função index do controller

Consultando um produto específico
GET
localhost:8000/api/produtos/53
Associada a função show do controller

Inserir um novo produto
POST
localhost:8000/api/produtos
Associada a função store do controller
Deve ser enviado pelo cliente json do novo produto

Alterar um produto
PUT
localhost:8000/api/produtos/53
Associada a função update do controller
Deve ser enviado pelo cliente json com os dados alterados

Excluir um produto
DELETE
localhost:8000/api/produtos/53
Associada a função destroy do controller
